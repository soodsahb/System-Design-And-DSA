1. Single Responsibility Principle (SRP)

Definition:
A class should have only one reason to change, meaning it should be responsible for only one functionality.

❌ Violation
class UserService {
  createUser(user) {}
  sendWelcomeEmail(user) {}
}

✅ Correct
class UserService {
  createUser(user) {}
}

class EmailService {
  sendWelcomeEmail(user) {}
}
